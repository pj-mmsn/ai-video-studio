# 长篇小说上下文管理方案

## 问题

100万字小说，LLM 上下文窗口（128K token ≈ 10万字）放不下。简单截断会导致：
- 角色性格前后矛盾
- 伏笔丢失
- 世界观规则被遗忘
- 章节间情节断层

## 方案：分层大纲 + 智能上下文检索

### 核心思路

**不是"把所有上下文喂给 LLM"，而是"只喂当前写作需要的上下文"。**

### 数据结构

```
卷 (Volume) 1-5
 └─ 章 (Chapter) 10-20/卷
      └─ 节 (Section) 3-8/章  ← LLM 一次写一节(1000-3000字)
           └─ 本次写作上下文 ≈ 2000-5000 tokens
```

### 数据库表设计

```
┌─────────────────────────────────────────────────────┐
│ novels           │ 小说元数据(标题/类型/状态)        │
│ outline_nodes    │ 分层大纲(卷→章→节, parent_id)    │
│ sections         │ 已写内容+版本                    │
│ characters       │ 角色档案(姓名/性格/弧线)          │
│ character_scenes │ 角色-场景关联(谁在哪节出场)       │
│ world_rules      │ 世界观(规则/地点/时间线)          │
│ plot_threads     │ 伏笔/剧情线(状态/关联章节)        │
│ context_logs     │ 每次写作用的上下文快照(调试用)     │
└─────────────────────────────────────────────────────┘
```

### 上下文检索策略

写第3卷第12章第2节时，从数据库精确查询：

| 取什么 | 怎么取 | 大约token |
|--------|--------|:--:|
| 本节大纲 | outline_nodes WHERE id=当前节 | 200 |
| 父级上下文 | 所属章大纲+卷大纲 | 300 |
| 出场角色 | 当前章关联的角色档案（浓缩版） | 500 |
| 活跃伏笔 | plot_threads WHERE status=open AND 关联本章 | 300 |
| 世界观规则 | world_rules 关联本章场景标签 | 200 |
| 前3节摘要 | sections 最近3节的摘要(非全文!) | 400 |
| 角色关系图 | 出场角色之间的关系摘要 | 200 |
| **合计** | | **~2100 tokens** |

### 上下文注入模板

```
## 当前位置
第{volume}卷第{chapter}章第{section}节
章节大纲: {outline_text}

## 前情提要
{last_3_sections_summary}

## 出场角色(请严格保持一致性)
{characters_in_scene}

## 活跃剧情线(请推进或收束)
{open_plot_threads}

## 世界观约束
{world_rules_for_scene}

---
请写本节内容(1000-3000字)。写完后自动提取: 新出场角色/新伏笔/世界观补充。
```

### 写作后自动提取

每节写完，LLM 自动提取并写入数据库：
- 本节出场的角色（新增或更新）
- 新埋的伏笔/已收束的伏笔
- 新揭示的世界观规则
- 本节摘要（2-3句话，供后续上下文检索用）

### 一致性保障

| 机制 | 怎么做 |
|------|--------|
| 角色一致性 | 每次注入角色的姓名+特征+首次出场章节，LLM 不敢乱改 |
| 伏笔追踪 | plot_threads 表记录 status(open/resolved)，未收束的永远注入 |
| 世界观约束 | world_rules 按场景标签注入，同场景规则不会冲突 |
| 情节连贯 | 前3节摘要保证相邻章节衔接顺畅 |
| 版本回滚 | sections 有 version 字段，每次修改增版本，可回退 |
